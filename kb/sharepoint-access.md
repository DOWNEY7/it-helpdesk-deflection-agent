# SharePoint Access

## Overview
SharePoint is used for team document libraries, intranet pages, and collaborative workspaces. This article covers access requests and troubleshooting.

## Requesting SharePoint Site Access

**For a team site:**
1. Navigate to the SharePoint site URL and click **Request access**
2. Add a justification message
3. The **site owner** (usually your team lead) will approve or deny the request
4. You will receive an email notification

**If you do not know the site URL:**
- Go to **https://company.sharepoint.com** → browse the intranet
- Or ask your team lead for the direct link

**IT cannot grant SharePoint access** — only site owners can. Contact the site owner directly.

## Troubleshooting Access Issues

### "Access Denied" on a site you previously accessed
1. Your account may have been removed from the site — contact the site owner
2. Check you are signed in with your **work account**, not a personal Microsoft account
3. Clear browser cache and cookies, then retry

### SharePoint site is slow or not loading
1. Try a different browser (Edge recommended for SharePoint)
2. Disable browser extensions (especially ad blockers)
3. Check your internet connection — SharePoint requires a stable connection
4. On VPN: ensure SharePoint traffic is not blocked (check with IT if using a proxy)

### Can't edit a document on SharePoint
1. Check-out: the file may be checked out by another user — look for the ✓ icon
2. Read-only permissions: you may have Visitor access only — ask site owner for Contributor access
3. File limit: maximum 250GB per file, 400-character path limit

### Syncing SharePoint Libraries to OneDrive
1. Navigate to the SharePoint library in your browser
2. Click **Sync** in the top toolbar
3. A prompt will open OneDrive — click **Start sync**
4. The library will appear under **SharePoint** in File Explorer

### "This site can't be reached" or "The site has been deleted"
- The SharePoint site may have expired or been archived
- Contact the site owner or your IT Service Desk
- IT can restore deleted sites within **93 days** of deletion

## File Sharing from SharePoint

To share a file with external users:
1. Right-click the file → **Share**
2. Click the link settings (Anyone / Specific people / Organisation)
3. External sharing must be enabled for the site — contact IT if this option is greyed out
4. Set an expiry date for external links

## When to Escalate
- Entire SharePoint site is inaccessible to all users (site down)
- Site was accidentally deleted more than 93 days ago
- Large-scale migration of files into SharePoint needed
